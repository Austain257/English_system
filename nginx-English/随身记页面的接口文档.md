# 英语知识积累本 - 接口文档

## 概述
本文档详细描述了英语知识积累本页面的API接口设计，包括接口功能、请求参数、响应格式和错误处理。

## 基础信息
- 基础URL：`http://localhost:8080/jotting`
- 请求格式：JSON
- 响应格式：JSON

## 接口列表

### 1. 获取知识列表
- **接口描述**：获取用户的所有英语知识记录
- **请求方式**：GET
- **接口地址**：`/list`
- **请求参数**：
  | 参数名 | 类型 | 必填 | 说明 |
  |--------|------|------|------|
  | page | int | 否 | 页码，默认为1 |
  | size | int | 否 | 每页数量，默认为10 |

- **响应示例**：
```json
{
  "code": 1,
  "msg": "success",
  "data": {
    "list": [
      {
        "id": 1,
        "english": "Hello world",
        "chinese": "你好，世界",
        "createTime": "2023-07-01 12:00:00",
        "reviewed": false
      }
    ],
    "total": 100
  }
}
```

### 3. 更新知识记录
- **接口描述**：更新已有的英语知识记录
- **请求方式**：PUT
- **接口地址**：`/update`
- **请求参数**：
  | 参数名 | 类型 | 必填 | 说明 |
  |--------|------|------|------|
  | id | int | 是 | 记录ID |
  | english | string | 否 | 英文内容 |
  | chinese | string | 否 | 中文释义 |
  | reviewed | boolean | 否 | 复习状态 |

- **请求示例**：
```json
{
  "id": 1,
  "english": "Updated English",
  "chinese": "更新的中文",
  "reviewed": true
}
```

- **响应示例**：
```json
{
  "code": 1,
  "msg": "更新成功",
  "data": null
}
```

### 4. 删除知识记录
- **接口描述**：删除指定的英语知识记录
- **请求方式**：DELETE
- **接口地址**：`/delete/{id}`
- **请求参数**：
  | 参数名 | 类型 | 必填 | 说明 |
  |--------|------|------|------|
  | id | int | 是 | 记录ID，通过路径参数传递 |

- **响应示例**：
```json
{
  "code": 1,
  "msg": "删除成功",
  "data": null
}
```

### 5. 批量删除知识记录
- **接口描述**：批量删除英语知识记录
- **请求方式**：POST
- **接口地址**：`/batchDelete`
- **请求参数**：
  | 参数名 | 类型 | 必填 | 说明 |
  |--------|------|------|------|
  | ids | array | 是 | 记录ID数组 |

- **请求示例**：
```json
{
  "ids": [1, 2, 3]
}
```

- **响应示例**：
```json
{
  "code": 1,
  "msg": "批量删除成功",
  "data": null
}
```

## 错误码说明

| 错误码 | 说明 |
|--------|------|
| 1 | 成功 |
| 0 | 失败 |
| -1 | 参数错误 |
| -2 | 记录不存在 |
| -3 | 权限不足 |
| -4 | 服务器内部错误 |

## 注意事项

1. 所有API请求都需要处理错误情况
2. 考虑添加请求超时处理
3. 对于重要操作（如删除）需要添加确认提示
4. 考虑添加加载状态提示
5. 根据实际业务需求，可能需要添加用户认证和权限控制

这个接口文档为英语知识积累本页面提供了完整的后端API设计，前端可以根据这些接口进行相应的修改，实现数据的远程存储和管理。
